所有权

管理堆上的数据，清理不再使用的垃圾

规则：
Rust 中的每一个值都有一个被称为其 所有者（owner）的变量。
值在任一时刻有且只有一个所有者。
当所有者（变量）离开作用域，这个值将被丢弃。

基础类型存储在栈上并且在离开作用域时被移出栈
String存储在堆上，可以由from函数创建
let s = String::from(“hello”)

移动
let s1 = String::from(“hello”)
let s2 = s1
s1赋给s2并不会有两个值一样的变量，s1会被认为不再有效，s1在栈上的数据传递给了s2
println!(“{}”,s1)会报错

克隆
需要复制堆上的数据，可以使用clone方法
let s1 = String::from(“hello”);
let s2 = s1.clone();
println!(“s1 = {}, s2 = {}”, s1, s2);

let x = 5;
let y = x;
println!(“x = {}, y = {}”, x, y);
整形这样存储在栈上的数据可以被快速复制，不需要clone

所有权与方法
变量传入函数时与变量赋值类似，也有移动，基础类型会被复制
函数返回值也会有移动，和传入类似

引用
为了避免所有权转移到函数里，可以使用引用来调用参数，也被称作借用
&String和&s1表示变量的引用，可以使用变量值，但是没有所有权
let len = calculate_length(&s1)
fn calculate_length(s: &String) -> usize {
s.len()
}

可变引用
引用默认不能修改值，需要改变值就要使用可变引用
fn main() {
let mut s = String::from(“hello”);

change(&mut s);

}

fn change(some_string: &mut String) {
some_string.push_str(“, world”);
}
但是在特定作用域中的特定数据只能有一个可变引用，以下代码会编译出错，rust在编译阶段就避免了数据竞争
let r1 = &mut s;
let r2 = &mut s;

引用与可变引用共存
引用的作用域是声明引用到最后一次使用之间，在最后一次使用之后声明使用可变引用是可以通过编译的，在引用作用域内声明可变引用会出错，可变引用可能会改变引用的值
let r1 = &s; // 没问题
let r2 = &s; // 没问题
let r3 = &mut s; // 大问题
println!(“{}, {}, and {}”, r1, r2, r3);

let mut s = String::from("hello");
let r1 = &s; // 没问题
let r2 = &s; // 没问题
println!("{} and {}", r1, r2);
// 此位置之后 r1 和 r2 不再使用
let r3 = &mut s; // 没问题
println!("{}", r3);

悬垂引用
内存在引用之前被释放，引用就会变成悬垂引用，编译器会在编译时避免这种错误，以下代码不会通过编译
fn dangle() -> &String {
let s = String::from(“hello”);
&s
}

slice
通过中括号指定的范围[0..5]创建一个slice，从零开始，包括第一个不包括最后一个
不写第一个从头开始，不写后一个到最后一个，都不写获取整个引用
let s = String::from(“hello world”);
let hello = &s[0..5];
let world = &s[6..11];

使用slice时更改字符串会报错，改变字符串会在底层使用可变引用，与不可变引用会冲突
let mut s = String::from(“hello world”);
let word = first_word(&s);
s.clear(); // 错误!
println!(“the first word is: {}”, word);

slice是一个不可变引用，字符串字面量就是指向二进制文件特定位置的slice，所以字符串字面量不可变

结构体

相当于别的语言里的类
声明一个结构体，结构体名，字段名，字段类型
struct User {
username: String,
email: String,
sign_in_count: u64,
active: bool,
}

创建一个结构体实例，如果需要可变需要加mut
let [mut] user1 = User {
email: String::from(“someone@example.com“),
username: String::from(“someusername123”),
active: true,
sign_in_count: 1,
}

获取和修改结构体的值
let mail = user1.email
user1.email = String::from(“anotheremail@example.com“)

函数最后一行构造返回结构体
fn build_user(email: String, username: String) -> User {
User {
email: email,
username: username,
active: true,
sign_in_count: 1,
}
}

变量名与字段名相同时可以简写，省略变量名
fn build_user(email: String, username: String) -> User {
User {
email,
username,
active: true,
sign_in_count: 1,
}
}

通过改变旧结构体创建新结构体，用user1的值填充为定义的字段
let user2 = User {
email: String::from(“another@example.com“),
username: String::from(“anotherusername567”),
..user1
};

定义元组结构体，可以给一种元组取一个名字，成为一种特定类型
struct Point(i32, i32, i32)
let black = Color(0, 0, 0)
与元组类似，可以结构，可以加点和索引获取元素

方法
声明结构体
struct rectangles {
width:u32,
height:u32,
}
声明方法，多个方法在一个imp块中和多个impl块中效果相同
impl rectangles{
fn area(&self) -> u32{
self.height * self.width
}
}
调用方法
let rec = rectangles{
width:4,
height:5,
};
println!(“{}”,rec.area());

关联函数
impl块中可以声明没有self参数的函数，作为这个结构体的关联函数，用::调用，常作为构造函数，例如String::from(“asd”)
