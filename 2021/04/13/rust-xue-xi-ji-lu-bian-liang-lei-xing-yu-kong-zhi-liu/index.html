rust学习记录-变量类型与控制流

rust
rust
发布日期:   2021-04-13

声明

声明变量前加mut代表可变变量
let spaces = “abc”;
let mut spaces = “abc”;
const声明常量，变量类型可以自动识别也可以加引号和类型名来声明
数字可以添加下划线增加可读性
MAX_POINTS: u32 = 100_000

重复声明可以隐藏变量，可以更改变量类型，值
let spaces = “abc”;
let spaces = spaces.len();

标量类型

整数

长度 有符号 无符号
8-bit i8 u8
16-bit i16 u16
32-bit i32（默认） u32
64-bit i64 u64
128-bit i128 u128
arch（64位机器就是64bit，32位机器就是32bit） isize usize
除 byte 以外的所有数字字面值允许使用类型后缀，例如 57u8
同时也允许使用 _ 做为分隔符以方便读数，例如1_000
进制
数字字面值 例子
Decimal (十进制) 98_222
Hex (十六进制) 0xff
Octal (八进制) 0o77
Binary (二进制) 0b1111_0000
Byte (单字节字符)(仅限于u8) b’A’

浮点

f32，f64（默认）

布尔

true，false

字符

char
大小为四个字节
单引号表示
let c =’z’
在 Rust 中，拼音字母，中文、日文、韩文等字符，emoji（绘文字）以及零长度的空白字符都是有效的 char 值

复合类型

元组

长度固定，一旦声明长度不会增大或减小
声明
let tup: (i32, f64, u8) = (500, 6.4, 1)
一个位置一个类型，不一定相同
获取
可以使用模式匹配来解构元组值获取
let tup = (500, 6.4, 1);
let (x, y, z) = tup;
println!(“The value of y is: {}”, y);
跟索引获取
let x: (i32, f64, u8) = (500, 6.4, 1);
let five_hundred = x.0;

数组

长度固定，一旦声明长度不会增大或减小
类型需要相同
声明
let a = [1, 2, 3, 4, 5]
let a: [i32; 5] = [1, 2, 3, 4, 5]
let a = [3; 5];
获取
let a = [1, 2, 3, 4, 5];
let first = a[0];
let second = a[1];

函数

Rust 不关心函数定义于何处，只要定义了就行
参数格式
fn another_function(x: i32, y: i32) {
println!(“The value of x is: {}”, x);
println!(“The value of y is: {}”, y);
}
返回类型与值
在括号后定义返回类型
fn five() -> i32 {
5
}
fn plus_one(x: i32) -> i32 {
x + 1
}

函数的返回值等同于函数体最后一个表达式的值，表达式没有结尾的分号
使用 return 关键字和指定值，可从函数中提前返回

控制流

if

条件必须是bool
if number < 5 {
println!(“condition was true”);
} else if number > 5 {
println!(“condition was false”);
}else{
println!(“something”);
}

if是表达式，可以用于赋值，但是值的类型需要相同，因为rust的需要在编译时就知道返回类型
let condition = true;
let number = if condition {
5
} else {
6
};

loop

无限循环，break可以退出循环，并将值作为循环的返回值
let mut i=0;
let y = loop {
i=i+1;
if i==10 {
break i*2
}
};
println!(“{}”,y);

while

let mut i=0;

while i<9 {
i=i+1;
println!(“{}”,i);
}
for
let numbers:[i32;5]=[1,2,3,4,5];
for number in numbers.iter(){
println!(“{}”,number)
}

可以使用(1..4)产生包含1不包含4的数字，rev可以让序列反转
for num in (1..4).rev(){
    println!("{}",num);
}
