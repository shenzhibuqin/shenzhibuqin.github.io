rust学习记录-泛型

定义泛型函数（不能编译）

fn largest<T>(list: &[T]) -> T {
    let mut largest = list[0];
    for &item in list.iter() {
        if item > largest {
            largest = item;
        }
    }
    largest
}

定义泛型结构体

struct Point<T>{
    x: T,
    y: T,
}
let integer = Point { x: 5, y: 10 };

定义泛型方法

impl<T> Point<T> {
    fn x(&self) -> &T {
        &self.x
    }
}

为特定类型定义方法

impl Point<f32> {
    fn distance_from_origin(&self) -> f32 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}

定义方法时泛型可以有impl声明也可以由方法声明，T，U与本身结构体定义对应，V，W是方法定义的泛型

struct Point<T, U> {
    x: T,
    y: U,
}

impl<T, U> Point<T, U> {
    fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W> {
        Point {
            x: self.x,
            y: other.y,
        }
    }
}

枚举泛型
option就是泛型的枚举类型，result也是

enum Option<T> {
    Some(T),
    None,
}

rust编译器在编译时确定泛型的具体类型，类似于下面这种变化，将option泛型变为具体类型，减小了运行时开销，速度和手写的重复代码类似，运行时效率极高

enum Option_i32 {
    Some(i32),
    None,
}

enum Option_f64 {
    Some(f64),
    None,
}

fn main() {
    let integer = Option_i32::Some(5);
    let float = Option_f64::Some(5.0);
}

trait
类似接口，规范泛型类的行为
use的引用规则和模块方法等类似

定义trait
多个方法的trait，一行一个方法签名，并以分号结尾

pub trait Summary {
    fn summarize(&self) -> String;
}

指定默认方法的trait实现

pub trait Summary {
    fn summarize(&self) -> String {
        String::from("(Read more...)")
    }
}

trait中的默认方法可以调用没实现的其他trait方法

pub trait Summary {
    fn summarize_author(&self) -> String;
    fn summarize(&self) -> String {
        format!("(Read more from {}...)", self.summarize_author())
    }
}

实现trait，或重载默认实现

impl Summary for NewsArticle {
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}

使用默认方法实现trait，一个空的impl块
impl Summary for NewsArticle {}

以trait为参数
impl指定trait方式

fn print_summary(&item: impl Summary){
    println!("Author:{}",item.summarize());
}

泛型方式

fn print_t_summary<T:Summary>(&item: T){
    println!("Author:{}",item.summarize());
}

通过+指定多个trait

fn print_summary(&item: impl Summary + Display){
    println!("Author:{}",item.summarize());
}
fn print_t_summary<T:Summary + Display>(&item: T){
    println!("Author:{}",item.summarize());
}

通过where从句简化泛型函数的trait指定

fn some_function<T: Display + Clone, U: Clone + Debug>(t: T, u: U) -> i32 {

可以变为

fn some_function<T, U>(t: T, u: U) -> i32
    where T: Display + Clone,
          U: Clone + Debug
{

使用trait bound为限制为泛型的结构体添加方法

struct Pair<T> {
    x: T,
    y: T,
}

impl<T> Pair<T> {
    fn new(x: T, y: T) -> Self {
        Self {
            x,
            y,
        }
    }
}

impl<T: Display + PartialOrd> Pair<T> {
    fn cmp_display(&self) {
        if self.x >= self.y {
            println!("The largest member is x = {}", self.x);
        } else {
            println!("The largest member is y = {}", self.y);
        }
    }
}

也可以对任何实现了特定 trait 的类型有条件地实现 trait。对任何满足特定 trait bound 的类型实现 trait 被称为 blanket implementations，他们被广泛的用于 Rust 标准库中。例如，标准库为任何实现了 Display trait 的类型实现了 ToString trait。这个 impl 块看起来像这样：

impl<T: Display> ToString for T {

生命周期

大生命周期引用小生命周期编译会失败

{
    let r;               // ---------+-- 'a
                          //               |
    {                     //               |
        let x = 5;    // -+-- 'b   |
        r = &x;       //  |            |
    }                     // -+          |
                          //               |
    println!("r: {}", r); //          |
}                         // ---------+

小生命周期可以引用大生命周期

#![allow(unused)]
fn main() {
{
    let x = 5;             // ----------+-- 'b
                               //                 |
    let r = &x;           // --+-- 'a    |
                               //     |           |
    println!("r: {}", r); //     |           |
                               // --+           |
}                              //  ----------+

泛型生命周期参数

方法的泛型生命周期参数表明返回引用的生命周期和参数的关系，让编译器可以检查
函数或方法的参数的生命周期被称为 输入生命周期（input lifetimes），而返回值的生命周期被称为 输出生命周期（output lifetimes）。

‘a代表的是string1和string2的重叠部分，返回参数的’a代表返回的引用生命周期等于string1和string2中短的那一个

let string1=String::from("this is longer");
let result;
{
    let string2 = String::from("this is shot");
    result =longest_s(&string1,&string2);
    println!("longer:{}",result);

}

fn longest_s<'a>(string1:& 'a str,string2:& 'a str)->&'a str{
    if string1.len()>string2.len(){
        string1
    }else {
        string2
    }
}

只在string1加’a就代表string1的生命周期，返回的引用也是string1的生命周期

let string1=String::from("this is longer");
let result;
{
    let string2 = String::from("this is shot");
    result =longest_s(&string1,&string2);

}
println!("longer:{}",result);

fn longest_s<'a>(string1:& 'a str,string2:&str)->&'a str{
    if string1.len()>string2.len(){
        string1
    }else {
        string1
    }
}

结构体的参数意味着结构体实例不能比其字段中的引用存在的更久

struct TestStruct<'a>{
    pub s: &'a str,
}

let s =String::from("asd");

let mut test = TestStruct {
    s:&s,

生命周期省略规则
符合特定模式时无需标注生命周期参数

编译器根据三条规则计算生命周期，计算完后没有得出生命周期就会报错
规则：

	每一个输入的引用类型参数都有自己的生命周期参数
	如果只有一个输入生命周期参数，则把它赋给所有输出
	如果有多个输入，且其中一个是self，说明是一个对象的方法，所有输出生命周期参数被赋予self的生命周期
```	
对于函数
一个输入一个输出的方法会把输入的生命周期给输出

对于方法
结构体字段的生命周期必须总是在 impl 关键字之后声明并在结构体名称之后被使用，

impl<’a> ImportantExcerpt<’a> {
fn announce_and_return_part(&self, announcement: &str) -> &str {
println!(“Attention please: {}”, announcement);
self.part
}
}


静态生命周期
static，生命周期能够存活于整个程序期间
所有字符串字面值都有static生命周期，标注出来就是下面这样
let s: &'static str = "I have a static lifetime.";
