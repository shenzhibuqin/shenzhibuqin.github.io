rust学习记录-cargo

rust
rust
发布日期:   2021-04-28

编译配置

cargo build
编译，dev模式
cargo build –release
release模式

可以在cargo.toml中修改编译的配置
[profile.dev]
opt-level = 0
[profile.release]
opt-level = 3
opt-level是优化程度，0-3，dev模式优化最低

文档注释

文档注释使用三斜杠///，内容采用md格式

/// Adds one to the number given.
///
/// # Examples
///
/// ```
/// let arg = 5;
/// let answer = my_crate::add_one(arg);
///
/// assert_eq!(6, answer);
/// ```
pub fn add_one(x: i32) -> i32 {
    x + 1
}

cargo doc –open
生成crate文档，并打开
examples是示例代码，cargo test也会测试（未成功）
除了example还可以有：
Panics：
这个函数可能会 panic! 的场景。并不希望程序崩溃的函数调用者应该确保他们不会在这些情况下调用此函数。
Errors：
如果这个函数返回 Result，此部分描述可能会出现何种错误以及什么情况会造成这些错误，这有助于调用者编写代码来采用不同的方式处理不同的错误。
Safety：
如果这个函数使用 unsafe 代码（这会在第十九章讨论），这一部分应该会涉及到期望函数调用者支持的确保 unsafe 块中代码正常工作的不变条件（invariants）。

包注释采用//!

//! # My Crate
//!
//! `my_crate` is a collection of utilities to make performing certain
//! calculations more convenient.

一般在包文件或lib等文件，为包或模块提供文档

一般在模块文件里可以用pub use将函数等重新导出，避免使用复杂路径引用
重导出也会在文档中体现

发布crate包

cargo publish
发布包
发布前要在cargo.toml添加一些信息
[package]
name = “guessing_game”
需要有一个和crate.io中已有的库不重名的名字
license = “MIT”
license标识符

发布的crate无法修改或删除
可以撤回，让新的项目不能引用你的库，撤回前已使用撤回库的老项目不影响使用
cargo yank –vers 1.0.1
撤销撤回
cargo yank –vers 1.0.1 –undo

工作空间

可以整合多个crate共同开发
根目录有定义工作空间的toml文件

[workspace]

members = [
    "crate1",
     "crate2",
]

创建crate
cargo new crate1
cargo new crate2 –lib

导入等和单个crate相同，通过toml导入本地crate
[dependencies]
add-one = { path = “../add-one” }
但是项目公用根目录的cargo.lock，保证环境依赖版本相同
编译输出也共用根目录的target
测试也会测试整个工作空间，或者cargo test -p 包名，测试单个包
要发布crate只能一个crate一个crate发

cargo install
可以用install命令安装编译好的二进制文件到~/.cargo/bin
设置环境就可以直接在命令行使用
