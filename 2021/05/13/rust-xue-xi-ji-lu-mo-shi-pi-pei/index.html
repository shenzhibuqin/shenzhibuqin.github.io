rust学习记录-模式匹配

rust
rust
发布日期:   2021-05-13

使用模式匹配的位置

最常用的位置就是match表达式，值，匹配的模式，要运行的表达式

match VALUE {
    PATTERN => EXPRESSION,
    PATTERN => EXPRESSION,
    PATTERN => EXPRESSION,
}

for循环也可以使用模式

在 for 循环中，模式是 for 关键字直接跟随的值，正如 for x in y 中的 x
通过for解构元组

let v = vec!['a', 'b', 'c'];
	
	for (index, value) in v.iter().enumerate() {
	    println!("{} is at index {}", value, index);
	}

let也使用模式

let PATTERN = EXPRESSION;
类似于let x =5；
变量名只不过是一个特别朴素的模式
我们将表达式与模式比较，并未任何找到的名称赋值
所以let可以使用模式解构元组

let (x, y, z) = (1, 2, 3);

如果模式不匹配会得到一个编译时错误

let (x, y) = (1, 2, 3);

可以考虑使用占位符_或..

let (x, y, _) = (1, 2, 3);
let(x,..)=(1,2,3);

函数参数也是模式

fn foo(x: i32){
}

x就是一个模式，类似let
在函数参数中用模式结构元组的例子

fn print_coordinates(&(x, y): &(i32, i32)) {
	    println!("Current location: ({}, {})", x, y);
	}
	
	fn main() {
	    let point = (3, 5);
	    print_coordinates(&point);
	}

类似于函数，闭包的参数也是模式匹配

可反驳与不可反驳

不可反驳：
能匹配任何传递的可能值的模式
如let x = 5
x可以匹配任何值
可反驳：
对可能的值进行匹配会失败的模式
如let Some(x) = a_value
如果a_value是None匹配会失败

函数参数、let语句、for循环只接受不可反驳的匹配，因为不匹配程序无法继续执行
if let和while let需要可反驳模式，不可反驳也能运行，编译器会警告但不会报错

匹配语法或用法

匹配多个模式

|可以用来匹配多个模式，相当于或操作

let x = 1;
	match x {
	    1 | 2 => println!("one or two"),
	    3 => println!("three"),
	    _ => println!("anything"),
	}

匹配范围

..=可以用来匹配一个数字或char范围
数字的例子

let x = 5;
	match x {
	    1..=5 => println!("one through five"),
	    _ => println!("something else"),
	}

char的例子

let x = 'c';
match x {
    'a'..='j' => println!("early ASCII letter"),
    'k'..='z' => println!("late ASCII letter"),
    _ => println!("something else"),
}

在匹配模式中创建值时检查值的匹配模式

@运算符可以检查在匹配模式中创建的值是否符合@符后跟着的匹配模式
以下例子先创建变量id_variable并用@来检查该变量是否符合一个匹配模式
再用..=直接匹配枚举的字段值
最后匹配枚举类型并用变量获取字段值

enum Message {
    Hello { id: i32 },
}

let msg = Message::Hello { id: 5 };

match msg {
    Message::Hello { id: id_variable @ 3..=7 } => {
        println!("Found an id in range: {}", id_variable)
    },
    Message::Hello { id: 10..=12 } => {
        println!("Found an id in another range")
    },
    Message::Hello { id } => {
        println!("Found some other id: {}", id)
    },
}

忽略模式中的值

下划线_是匹配但不绑定任何值的通配符模式，可以用于match的最后一条来处理剩余情况
也可以在函数签名中使用，忽略不需要的参数

fn foo(_: i32, y: i32) {
    println!("This code only uses the y parameter: {}", y);
}

忽略模式中不需要的值

let mut setting_value = Some(5);
let new_setting_value = Some(10);
match (setting_value, new_setting_value) {
    (Some(_), Some(_)) => {
        println!("Can't overwrite an existing customized value");
    }
    _ => {
        setting_value = new_setting_value;
    }
}

..也可以忽略剩余值，用在忽略多个值的情况下，他会忽略剩余的没有显式匹配的值

struct Point {
    x: i32,
    y: i32,
    z: i32,
}
let origin = Point { x: 0, y: 0, z: 0 };
match origin {
    Point { x, .. } => println!("x is {}", x),
}
..忽略必须明确
(first, .., last)
上面这样不会报错
(.., second, ..)
上面这样不明确的忽略rust会报错

忽略对未使用变量的警告

声明了变量但未使用的情况下会有一个警告，用下划线开头来告诉编译器去掉警告
在准备目前使用不到的变量时可以用下划线开头

解构

模式可以解构结构体，枚举，元组，引用

解构结构体

解构一个结构体的例子

struct Point {
    x: i32,
    y: i32,
}
fn main() {
    let p = Point { x: 0, y: 7 };
    let Point { x: a, y: b } = p;
    assert_eq!(0, a);
    assert_eq!(7, b);
}

因为变量名与字段名经常相同

let Point { x: x, y: y } = p;

为了不重复可以简写

struct Point {
    x: i32,
    y: i32,
}
fn main() {
    let p = Point { x: 0, y: 7 };
    let Point { x, y } = p;
    assert_eq!(0, x);
    assert_eq!(7, y);
}

也可以指定一部分来匹配特定情况

let p = Point { x: 0, y: 7 };
    match p {
        Point { x, y: 0 } => println!("On the x axis at {}", x),
        Point { x: 0, y } => println!("On the y axis at {}", y),
        Point { x, y } => println!("On neither axis: ({}, {})", x, y),
    }

解构枚举

解构枚举需要对应枚举值存储的数据

enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    ChangeColor(i32, i32, i32),
}
fn main() {
    let msg = Message::ChangeColor(0, 160, 255);
    match msg {
        Message::Quit => {
            println!("The Quit variant has no data to destructure.")
        }
        Message::Move { x, y } => {
            println!(
                "Move in the x direction {} and in the y direction {}",
                x,
                y
            );
        }
        Message::Write(text) => println!("Text message: {}", text),
        Message::ChangeColor(r, g, b) => {
            println!(
                "Change the color to red {}, green {}, and blue {}",
                r,
                g,
                b
            )
        }
    }
}

解构嵌套的复杂结构

解构嵌套的元组

let ((feet, inches), Point {x, y}) = ((3, 10), Point { x: 3, y: -10 });

解构嵌套的结构体和枚举

enum Color {
   Rgb(i32, i32, i32),
   Hsv(i32, i32, i32),
}

enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    ChangeColor(Color),
}

fn main() {
    let msg = Message::ChangeColor(Color::Hsv(0, 160, 255));

    match msg {
        Message::ChangeColor(Color::Rgb(r, g, b)) => {
            println!(
                "Change the color to red {}, green {}, and blue {}",
                r,
                g,
                b
            )
        }
        Message::ChangeColor(Color::Hsv(h, s, v)) => {
            println!(
                "Change the color to hue {}, saturation {}, and value {}",
                h,
                s,
                v
            )
        }
        _ => ()
    }
}
